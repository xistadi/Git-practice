git config 
git init
git clone
git add - добавляет содержимое рабочей директории в индекс 
(staging area) для последующего коммита
git commit - берёт все данные, добавленные в индекс с помощью 
git add, и сохраняет их слепок во внутренней базе данных, 
а затем сдвигает указатель текущей ветки на этот слепок
git diff - вычислениe разницы между любыми двумя Git деревьями
git reset - отмена изменений
git status - показывает состояния файлов в рабочей директории и 
индексе
git rm - используется в Git для удаления файлов из индекса и 
рабочей директории
git log - просмотр истории коммитов, начиная с самого свежего и 
уходя к истокам проекта
git show - просмотр информации о метке или коммите
git tag - заданиe постоянной метки на какой-либо момент в 
истории проекта. Обычно она используется для релизов.
git branch - это своего рода “менеджер веток”. Она умеет 
перечислять ваши ветки, создавать новые, удалять и 
переименовывать их.
git checkout - используется для переключения веток и выгрузки их
содержимого в рабочую директорию.
git merge - используется для слияния одной или нескольких веток в текущую
git remote - управлениe списком удалённых репозиториев
git push - вливание локальных изменений в удаленный репозиторий
git pull -вливание изменений из удаленного репозитория в
локальный
git stash - используется для временного сохранения всех 
незакоммиченных изменений для очистки рабочей директории 
без необходимости коммитить незавершённую работу в новую ветку.
